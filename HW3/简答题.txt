1. 精确率是什么？为什么不能只用它评估分类性能？
精确率 (Precision)：预测为正类的样本中，实际为正类的比例。
Precision = TP / (TP + FP)
（TP: 真阳性；FP: 假阳性）
为什么不能只用精确率？
忽略负类样本的表现（如漏检的负类）。
在样本不均衡时可能误导：
*例：99%负样本，1%正样本。若模型将所有样本预测为负，精确率=0/0（未定义），但实际模型完全失效。
需结合召回率 (Recall) 评估正类覆盖率。

2. 简述下混淆矩阵是什么
混淆矩阵（Confusion Matrix）是机器学习中用于评估分类模型性能的核心工具，它是分类模型预测结果的汇总表格，展示四类结果
		 实际为正类	  实际为负类
预测为正类		TP		        FP
预测为负类		FN			TN

3. 简述下各个性能指标的意思： 准确率，召回率，F1分数，假阳性，PR曲线，ROC曲线，AUC分数
准确率 (Accuracy)：(TP + TN) / (TP+FP+FN+TN)	整体正确率，但样本不均衡时不可靠
召回率 (Recall)	  ：TP / (TP + FN)	正类覆盖率（查全率）
F1 分数	          ：2 * (Precision*Recall) / (Precision+Recall)精确率与召回率的调和平均（平衡指标）
假阳性率 (FPR)	  ：FP / (FP + TN)	负类被误判为正类的比例
PR 曲线		  ：横轴：Recall；纵轴：Precision	展示不同阈值下 P-R 的权衡（样本不均衡时更敏感）
ROC 曲线		  ：横轴：FPR；纵轴：Recall (TPR)	展示分类器泛化能力（对角线=随机猜测）
AUC 分数	          ：ROC曲线下面积	值越接近1模型越好；0.5=无判别力

4. 简述下准确率-召回率权衡
高准确率意味着模型预测为正例的结果很可靠，误报（FP）很少，高召回率意味着模型找全正例的能力强，漏报（FN）很少。准确率-召回率权衡主要看假正例和假反例带来的实际成本或风险哪个更高。
FP成本高： 优先保证高准确率 (e.g., 法律判决、金融风控、推荐系统误推)。
FN成本高： 优先保证高召回率 (e.g., 疾病筛查、安全漏洞检测、欺诈检测漏网)。

5. 如何用二元分类器 去解决多元分类的问题
两种主流策略：
One-vs-Rest (OvR)：训练 K 个二元分类器（K=类别数）第 i 个分类器：将第 i 类 vs 其他所有类
预测依据：置信度最高的分类器结果
One-vs-One (OvO)：训练 C(K,2) 个二元分类器（每两类训练一个）
预测依据：投票制，胜出最多的类别为最终结果


6. 什么是 多标签-多分类问题？
一个样本同时属于多个标签/类别（非互斥）的分类问题。
核心特征：
输入： 一个样本（如图像、文本、数据点）。
输出： 一个标签子集（Label Subset）。这个子集可以是空集（不属于任何类别）、包含一个标签（等同于单标签情况）、或包含多个标签。
标签空间： 存在一个固定的、预定义的标签集合 L = {l₁, l₂, ..., lₖ}，其中 k 是总的类别数。
标签相关性： 标签之间可能存在依赖关系（相关性或互斥性），这是多标签问题的难点和重点之一。
解决策略与方法：

解决多标签问题主要有三大类策略：
一，问题转化（Problem Transformation）：
核心思想： 将多标签问题转化为一个或多个单标签分类问题（二分类或多分类），然后使用现有的单标签分类器（如SVM、决策树、逻辑回归、神经网络）来解决。
常用方法：
1)二元关联（Binary Relevance - BR），2)分类器链（Classifier Chains - CC），3)标签幂集（Label Powerset - LP）

二，算法适配（Algorithm Adaptation）：
核心思想： 直接修改或扩展现有的机器学习算法，使其能够原生地处理多标签数据，直接输出多个标签。
常用方法/算法：
1)多标签决策树/随机森林，2)多标签K近邻（ML-kNN）,3)多标签神经网络,4其他适配算法： 如适配的支持向量机（SVM）、贝叶斯方法等。

三，集成方法（Ensemble Methods）
核心思想： 结合多个基础多标签分类器（可以是问题转化型或算法适配型）的预测结果，以期获得比单个分类器更好的泛化性能和鲁棒性。旨在缓解单一方法（如BR忽略相关性，LP的数据稀疏）的缺点。
常用方法：
1)集成二元关联（Ensemble of Binary Relevance - EBR），2)随机k标签子集（Random k-Labelsets - RAKEL）,3)集成分类器链（Ensemble of Classifier Chains - ECC）,4)集成算法适配模型
